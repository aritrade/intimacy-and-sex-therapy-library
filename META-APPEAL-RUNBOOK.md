## STEP 4 — After submitting

1. **Record the case/reference number** for every appeal.
2. **Typical timelines:** account/Page reviews often resolve in **48h–2 weeks**;
   developer-platform reviews can take longer. Don't spam-resubmit — one clear
   appeal per channel.
3. **If you get a follow-up email**, reply promptly with the exact info requested
   (clinician credentials, ID verification, business documents). Slow replies are
   the #1 reason appeals stall.
4. **If denied**, you usually get one more review; escalate via the Business Help
   Center contact form with any new supporting docs. In parallel, start the
   **Rebuild path** below so you're not blocked on Meta's decision.

---

## Content positioning — do's & don'ts (applies to appeal AND future posts)

**Do:** frame everything as *sexual-health education*; emphasize *clinician review*
and *editorial moderation*; keep visuals clinical/illustrative; use precise,
educational language.

**Don't:** describe it as "adult content"; use explicit imagery/thumbnails; argue
that the policy is wrong; create multiple duplicate appeals from multiple accounts
(that itself is a violation).

## Rebuild path (run in parallel — no code deploy needed)

If your Page + IG are healthy but the dev/app layer stays blocked, rebuild the app
layer under a **clean Business portfolio** (a different admin profile in good
standing). Full click-by-click steps live in
[`RUNBOOK-CONTENT-ENGINE.md`](./RUNBOOK-CONTENT-ENGINE.md) ("Provision Instagram
Graph API"). Summary:

1. Create/verify a new Business portfolio → add Page 61590557572787 + the IG
   account to it.
2. Create a new app → add **Instagram Graph API** + **Facebook Login for Business**.
3. Create a **System User** → generate a **non-expiring** token with scopes:
   `instagram_basic`, `instagram_content_publish`, `pages_manage_posts`,
   `pages_read_engagement`, `pages_show_list`, `business_management`.
4. Submit **App Review** for `instagram_content_publish` (screencast + the
   clinical/educational justification from Template A). 3–14 days.

### Turning IG/FB back on once you have a valid token (5 minutes, no deploy)

1. In Vercel → Project → Settings → Environment Variables, set:
   - `META_GRAPH_ACCESS_TOKEN` = the new system-user token
   - `INSTAGRAM_BUSINESS_ACCOUNT_ID` = from `GET /{page-id}?fields=instagram_business_account`
   - `META_FACEBOOK_PAGE_ID` = `61590557572787`
2. Set `AUTO_PUBLISH_PLATFORMS=instagram,youtube,facebook`.
3. Redeploy (or wait for the next deploy), then run the **social-canary** workflow
   (Actions tab → social-canary → Run). When all three report `ok: true`, IG + FB
   automatically join the existing 6/day rollout — the verify-then-reclaim logic
   already covers them.
